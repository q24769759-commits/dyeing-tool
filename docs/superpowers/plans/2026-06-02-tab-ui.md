# Tab UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the library bar with a 檢視／寫入 tab bar so the user's current mode is visually explicit.

**Architecture:** Pure UI refactor of `index.html`. The existing JS functions `enterViewMode`, `enterDesignMode`, `loadProcess`, `fetchLibraryList`, and `saveToLibrary` are kept; only their internal DOM manipulation is trimmed to match the new structure. A new `switchTab(tab)` function drives tab switching. No changes to Apps Script or data flow.

**Tech Stack:** Vanilla HTML/CSS/JS (single file)

- [ ] **Step 1: Add switchTab function**

  Find the `// ── 初始化` comment. Add the following function immediately BEFORE it:

  ```javascript
  function switchTab(tab) {
    const isView = tab === 'view';
    const wasView = document.getElementById('tabView').classList.contains('tab-active');
    document.getElementById('tabView').classList.toggle('tab-active', isView);
    document.getElementById('tabWrite').classList.toggle('tab-active', !isView);
    document.getElementById('tabViewPanel').style.display = isView ? 'flex' : 'none';
    // Reset badge whenever entering 檢視 tab (shown only after a process is loaded)
    if (isView) document.getElementById('viewBadge').style.display = 'none';
    // Only clear table when switching FROM 檢視 TO 寫入 (not on page init)
    if (!isView && wasView) enterDesignMode();
  }
  ```
